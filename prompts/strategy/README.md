## Phân tích SWOT

Dùng cho: phân tích SWOT cho doanh nghiệp hoặc dự án.

```markdown
# Context:
Doanh nghiệp: [tên], ngành [ngành], quy mô [quy mô].
Phân tích cho: [mục đích cụ thể].

# Commission:
Phân tích SWOT.
- Format: S + W + O + T
- Mỗi phần: 3-5 điểm

# Content:
[Thông tin bổ sung nếu có]

# Criteria:
- Phải có: cả 4 phần đều có, mỗi phần 3-5 điểm cụ thể
- Không được: điểm generic không action được
- Dấu hiệu tốt: có thể chuyển thành action plan từ mỗi điểm
```

## So sánh suppliers

Dùng cho: so sánh 2-3 suppliers trước khi quyết định.

```markdown
# Context:
Cần tìm supplier cho [sản phẩm].
Đang cân nhắc: [list suppliers].

# Commission:
So sánh suppliers theo criteria đã định.
- Format: bảng so sánh + pros/cons
- Đánh giá theo: [list criteria]

# Content:
Suppliers:
- Supplier A: [thông tin]
- Supplier B: [thông tin]
- Supplier C: [thông tin]

Criteria quan trọng:
1. [criteria 1]
2. [criteria 2]
3. [criteria 3]

# Criteria:
- Phải có: rating theo từng criteria, pros/cons rõ ràng
- Không được: thiếu thông tin cụ thể, không đánh giá theo criteria
- Dấu hiệu tốt: có thể quyết định được sau khi đọc
```

## Research đối thủ

Dùng cho: nghiên cứu thông tin đối thủ.

```markdown
# Context:
Doanh nghiệp: [tên], ngành [ngành].
Muốn research đối thủ: [list đối thủ].

# Commission:
Research đối thủ.
- Output: profile ngắn mỗi đối thủ + so sánh

# Content:
Đối thủ:
- [Tên 1]: [thông tin đã biết]
- [Tên 2]: [thông tin đã biết]

Focus research:
- Sản phẩm/pricing
- Marketing strategy
- Customers
- Strengths/weaknesses

# Criteria:
- Phải có: thông tin chính xác, source rõ, comparison useful
- Không được: speculation không có basis, thiếu source
- Dấu hiệu tốt: có insights có thể áp dụng cho brand mình
```

## Kế hoạch tháng

Dùng cho: lên kế hoạch tháng/quý.

```markdown
# Context:
Team: [tên team], mảng [mảng].
Lên kế hoạch cho: [tháng/quý].

# Commission:
Viết kế hoạch tháng.
- Format: priorities + action items + KPIs
- Realistic với resources hiện có

# Content:
Budget: [số]
Team size: [số người]
Last month performance: [số liệu]

# Criteria:
- Phải có: 3-5 priorities rõ ràng, action items cụ thể, KPIs đo được
- Không được: quá nhiều priorities (>5), action items không khả thi
- Dấu hiệu tốt: team đọc xong biết phải làm gì tuần này
```

## Benchmark thị trường

Dùng cho: nghiên cứu xu hướng, benchmark thị trường.

```markdown
# Context:
Industry: [ngành], thị trường [VN/global].
Benchmark về: [chủ đề cụ thể].

# Commission:
Benchmark thị trường.
- Output: summary + key findings + implications

# Content:
Focus areas:
- [Area 1]
- [Area 2]

Companies to include: [list]

# Criteria:
- Phải có: data cụ thể, source rõ, implications cho business
- Không được: generic industry trends, không actionable
- Dấu hiệu tốt: có thể chuyển thành recommendation cụ thể
```
