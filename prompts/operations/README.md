## Email FAQ

Dùng cho: trả lời câu hỏi thường gặp từ khách hàng.

```markdown
# Context:
Shop bán [sản phẩm], khách hàng thường hỏi về [chủ đề].

# Commission:
Viết 1 email trả lời FAQ cho khách hàng.
- Tone: thân thiện, chuyên nghiệp
- Độ dài: dưới 150 chữ

# Content:
Câu hỏi khách: [paste câu hỏi]
Thông tin trả lời: [paste thông tin]
Shop: [tên shop, địa chỉ, hotline]

# Criteria:
- Phải có: lời chào, trả lời đúng câu hỏi, thông tin shop, lời kết
- Không được: quảng cáo trong email, hứa hẹn không đúng
- Dấu hiệu tốt: khách có thể tự trả lời được sau khi đọc
```

## Email xác nhận

Dùng cho: xác nhận đơn hàng, lịch hẹn, booking.

```markdown
# Context:
Shop [tên], bán [sản phẩm], giao hàng qua [kênh].

# Commission:
Viết email xác nhận cho khách.
- Format: subject + body
- Độ dài: dưới 100 chữ

# Content:
- Tên khách: [tên]
- Mã đơn: [mã]
- Sản phẩm: [danh sách]
- Tổng tiền: [số tiền]
- Dự kiến giao: [ngày]

# Criteria:
- Phải có: cảm ơn, thông tin đơn chính xác, timeline giao hàng, hotline
- Không được: sai thông tin sản phẩm/giá
- Dấu hiệu tốt: khách xác nhận đúng thông tin
```

## Báo cáo tuần

Dùng cho: tổng hợp số liệu tuần gửi manager.

```markdown
# Context:
Team [tên team], làm [mảng công việc], báo cáo cho [người nhận].

# Commission:
Viết báo cáo tuần theo format chuẩn.
- Format: overview + chi tiết + next week
- Độ dài: 1 trang A4

# Content:
Tuần: [số tuần], ngày [date]
Số liệu tuần này: [paste số liệu]
Số liệu tuần trước: [paste số liệu]
Mục tiêu tuần tới: [mục tiêu]

# Criteria:
- Phải có: so sánh với tuần trước, highlight điểm bất thường, mục tiêu tuần tới
- Không được: thiếu số liệu cụ thể, không so sánh
- Dấu hiệu tốt: manager đọc xong hiểu tình hình trong 2 phút
```

## Caption sản phẩm

Dùng cho: viết caption đăng sản phẩm hàng ngày.

```markdown
# Context:
Shop [tên], bán [sản phẩm], IG followers [số].
Khách hàng: [mô tả khách hàng].

# Commission:
Viết [số] caption Instagram cho sản phẩm.
- Mỗi caption: [số] chữ
- Có hook 1 dòng đầu
- Có CTA mua hàng

# Content:
Sản phẩm: [tên, giá, mô tả ngắn]
USP: [điểm bán độc nhất]
Kênh bán: [IG/TikTok Shop]

# Criteria:
- Phải có: tên sp, giá, USP, CTA
- Không được: so sánh với đối thủ, "sang trọng/upscale"
- Dấu hiệu tốt: khách hỏi mua sau khi đọc
```

## Meta description

Dùng cho: viết meta description cho SEO website.

```markdown
# Context:
Website bán [sản phẩm], domain [domain].

# Commission:
Viết [số] meta description.
- Độ dài: 150-160 ký tự
- Có keyword chính: [keyword]
- Có CTA: [CTA]

# Content:
Keyword chính: [keyword]
Keyword phụ: [keywords]
USP: [USP]

# Criteria:
- Phải có: keyword chính, USP ngắn gọn, CTA
- Không được: quá 160 ký tự, keyword stuffing
- Dấu hiệu tốt: click rate cao khi test
```

## Meeting notes

Dùng cho: tóm tắt nội dung meeting sau khi họp xong.

```markdown
# Context:
Meeting [tên/mục đích], ngày [date], tham gia [danh sách người].

# Commission:
Viết meeting notes tóm tắt.
- Format: attendees + agenda + decisions + action items
- Độ dài: dưới 1 trang

# Content:
[Dán nội dung meeting hoặc ghi chép]

# Criteria:
- Phải có: ai tham gia, quyết định gì, ai làm gì, deadline
- Không được: thiếu action items, không rõ ai responsible
- Dấu hiệu tốt: ai đọc notes đều biết mình phải làm gì
```

## SOP viết quy trình

Dùng cho: viết Standard Operating Procedure cho quy trình trong công ty.

```markdown
# Context:
Quy trình: [tên quy trình]
Team: [team phụ trách]
Áp dụng khi: [trigger/tình huống]

# Commission:
Viết SOP chi tiết.
- Format: purpose + steps + checkpoint
- Mỗi step: 1 action rõ ràng

# Content:
[Mô tả quy trình hiện tại nếu có]
[Các bước chính]

# Criteria:
- Phải có: mục đích, danh sách steps, checkpoint verify, người responsible
- Không được: step ambiguous (có thể hiểu nhiều cách), thiếu checkpoint
- Dấu hiệu tốt: người mới đọc SOP làm đúng được
```
